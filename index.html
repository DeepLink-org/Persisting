    Quick Start

    1

    Install Persisting

    pip install persisting

# Or install with Pulsing
pip install persisting[pulsing]
    2

    Register Backend with Pulsing

    import pulsing as pul
import persisting as pst

# Register Lance backend
pul.queue.register_backend("lance", pst.queue.LanceBackend)

# Or register enhanced Persisting backend
pul.queue.register_backend("persisting", pst.queue.PersistingBackend)
    3

    Use Persistent Queue

    import pulsing as pul

# Create queue with Lance persistence
writer = await pul.queue.write_queue(
    system, "my_topic",
    backend="lance",
    storage_path="/data/queues"
)

# Write data
await writer.put({"id": "1", "value": 42})
await writer.flush()  # Persist to disk

    Why Persisting?

    Persisting is a persistent storage solution tailored for AI systems. Built on the Lance columnar format, it provides high-performance data read/write capabilities while deeply integrating with the Pulsing Actor framework to provide reliable persistence for distributed queues. Whether it's KV Cache, Trajectories, or model parameters, Persisting handles them efficiently.

    Learn More ‚Üí

    Pluggable Storage Backends

    Persisting provides a flexible storage backend architecture. Choose Memory backend (for testing), Lance backend (for production), or implement custom backends based on your needs.

    All backends follow the unified StorageBackend protocol, ensuring API consistency. Switch backends with a single parameter change, no business logic modifications required.

    Learn More ‚Üí 
    # Switch between backends easily
import pulsing as pul
import persisting as pst

# Memory backend for testing
writer = await pul.queue.write_queue(system, "test",
    backend="memory")

# Lance backend for persistence
writer = await pul.queue.write_queue(system, "prod",
    backend="lance",
    storage_path="/data/queues")

# Enhanced backend with WAL
writer = await pul.queue.write_queue(system, "advanced",
    backend=pst.queue.PersistingBackend,
    backend_options={"enable_wal": True})
