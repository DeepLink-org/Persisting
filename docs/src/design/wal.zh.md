# Write-Ahead Log (WAL)

> **状态：规划中** — WAL 尚未实现。本文档描述未来实现的设计。

## 概述

Write-Ahead Log (WAL) 将通过在写入主存储之前先记录变更来保证数据持久性，使得崩溃后可以恢复未提交的数据。

## 设计目标

1. **崩溃恢复** — 已确认的写入不会丢失。
2. **低开销** — 对写入吞吐量的影响最小。
3. **实现简单** — 追加式日志 + 定期截断。

## 计划架构

```
put(record)
    │
    ├──► WAL 追加（顺序写入）
    │
    └──► 内存缓冲区
              │
              ▼ flush()
         Lance 数据集
              │
              ▼
         WAL 截断
```
